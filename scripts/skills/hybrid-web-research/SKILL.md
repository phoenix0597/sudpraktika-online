# Навык: Гибридный веб-поиск (экономия квоты MCP)

## Когда применять

Перед каждым обращением к MCP веб-поиску (WebSearch) или веб-чтению (webReader).
Эти инструменты исчерпывают месячную квоту ZCode (~10 вызовов = 79% квоты за сессию).

## Цепочка поиска (по убыванию приоритета)

**1. Оффлайн-знание через DeepSeek API** (бесплатно для квоты ZCode)

Для устойчивых фактов: значения enum, структуры API, отраслевые диапазоны (RPM, CPC),
практики SEO, конфигурации, форматы данных. DeepSeek знает многое из обучающей базы
без живого веба.

Запуск: `py scripts/hybrid_lookup.py "<вопрос>"` — см. скрипт ниже.

**2. Прицельный HTTP через Bash** (бесплатно для квоты)

Для конкретных URL: документация, Swagger, robots.txt, тестовые вызовы API.
Использует `requests`/`curl`, не MCP.

**3. MCP веб-поиск/чтение** (последнее средство)

Только когда 1 и 2 невозможны: нужен поиск по свежему контенту, которого нет в
обучающих данных, или URL невозможно получить напрямую. Максимум 1–2 за сессию.

## Что НЕ искать через DeepSeek (он не умеет)

- Свежие новости, последние изменения (после cutoff обучения).
- Содержимое конкретного живого URL (используйте HTTP/Bash).
- Субъективные оценки или нюансы текущей выдачи поисковиков.

## Доказанный кейс (2026-06-24)

Блокер: Yandex Search API требовал enum `SearchType`, сервер отвергал все догадки.
DeepSeek API сразу выдал точные значения (`SEARCH_TYPE_RU`, `SEARCH_TYPE_TR`, ...).
Результат: блокер снят без единого MCP-вызова. API подтверждён рабочим (HTTP 200).
