# START-HERE

Этот файл является стартовой точкой для возобновления работы над проектом в новой сессии.
Это индекс, а не канонический источник проектных правил или архитектурных решений.

## Сначала Прочитать

Если новая сессия начинается в этом проекте, прочитай эти файлы перед продолжением работы:

- Глобальные правила сотрудничества текущего пользователя: `.codex/memories/global-collaboration-rules.md` относительно домашнего каталога
- Глобальную архитектуру памяти текущего пользователя: `.codex/memories/memory-architecture.md` относительно домашнего каталога
- `AGENTS.md`
- `assistant-memory.md`
- `technical-specification-mvp.md`, если файл существует
- `memory/current-work-queue.md`, если файл существует
- `START-HERE.md`
- `memory/YYYY-MM-DD.md` за сегодня, если файл существует; затем последнюю недавнюю ежедневную заметку при необходимости

## Форматы Путей

Форматы путей и соглашения о них описаны в `AGENTS.md` этого проекта и в глобальной `.codex/memories/memory-architecture.md` относительно домашнего каталога; применяй их и здесь.

## Как Возобновлять Работу

1. Прочитай перечисленные выше файлы по порядку. Роли канонических файлов (`AGENTS.md`, `assistant-memory.md`, `technical-specification-mvp.md`, `memory/current-work-queue.md`, daily/topic-заметки) описаны в `AGENTS.md` этого проекта — применяй их.
2. Используй ежедневные файлы в `memory/` для свежего хронологического контекста и переходи по ссылкам из них в тематические файлы при необходимости.

## Рекомендуемый Следующий Шаг

- Не держи живую пошаговую очередь в этом файле.
- Используй `memory/current-work-queue.md` для текущего декомпозированного плана реализации и рекомендуемых следующих шагов.
- Оставляй этот файл только стартовым индексом.
