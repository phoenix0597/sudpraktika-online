# Задача: переобработать practice-разборы с неполным применением норм

Восстанови контекст проекта.

## Цель

Переобработать 13 опубликованных страниц дел, перечисленных в:

`data/review/practice-norm-application-debt.txt`

## Проблема

В соответствующих `data/structured/practice_<docid>.md` часть legacy-норм в разделе `Нормы, на которые сослался суд` содержит только общее пояснение нормы, но не содержит отдельного подпункта `Применение судом`.

Генератор не должен подставлять слабую заглушку. Нужно исправить именно исходные `practice_*.md`.

## Что нужно сделать

Для каждого `docid` из `data/review/practice-norm-application-debt.txt` открыть:

- `data/structured/practice_<docid>.md`
- `data/structured/structure_<docid>.json`
- `data/structured/user_story_<docid>.md`
- `data/raw_acts/act_<docid>.txt`

В `practice_<docid>.md` привести раздел `Нормы, на которые сослался суд` к каноническому формату:

```md
* **<норма>**
  * **Значение в деле:** <что означает норма для этого дела>
  * **Применение судом:** <как именно суд применил/учёл/отклонил эту норму в деле>
```

`Применение судом` писать только на основании судебного акта и уже имеющихся материалов дела. Не добавлять юридических выводов, которых нет в акте.

Если по норме невозможно уверенно определить применение, указать это нейтрально и конкретно, например:

```md
  * **Применение судом:** Суд привёл эту норму в составе правового обоснования, но отдельно её применение не раскрыл.
```

## Ограничения

- Не редактировать вручную `site_prototype/**/*.html`.
- После правки markdown пересобрать сайт генератором.
- Запрещено использовать внешние LLM API или API-скрипты без отдельного явного разрешения пользователя.
- Коммит не делать без отдельного подтверждения.

## Обновление debt-list

После исправления каждого `docid` обновить:

`data/review/practice-norm-application-debt.txt`

Правило:

- удалить строку, если долг по делу закрыт полностью;
- уменьшить счётчик, если долг закрыт частично.

Цель — закрыть все 13 строк, если это возможно без выдумывания содержания.

## Проверки в конце

Запустить:

```powershell
python scripts/check_practice_norm_application_debt.py
npm run test:data
python scripts/generate_ssg_prototype.py
npm run test:smoke
```

## Отчёт

В конце подготовить краткий отчёт:

- сколько `docid` исправлено;
- остался ли долг в `practice-norm-application-debt.txt`;
- какие проверки прошли;
- какие файлы изменены;
- есть ли случаи, где применение нормы невозможно уверенно восстановить по акту.
