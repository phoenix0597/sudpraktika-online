# DDU data layer

Данные судебной практики по ДДУ/долевому строительству хранятся отдельно от ЗоПП-корпуса, чтобы не смешивать очереди, сырые акты, структурированные JSON и реестры.

## Каталоги

- `raw_acts/` — очищенные тексты судебных актов `act_<docid>.txt` и метаданные `act_<docid>.meta.json`.
- `structured/` — результаты обработки: `user_story_<docid>.md`, `practice_<docid>.md`, `structure_<docid>.json`.
- `inbox/` — очередь для ИИ-агента: `_queue.json` и при необходимости локальные task-файлы.
- `registry/` — CSV-реестры известных дел для дедупликации.
- `parse_batches/` — рабочие каталоги партий парсинга.

## Цель уверенного старта

- 150–250 валидных дел.
- 8–12 страниц-ситуаций.
- 3–5 сильных страниц с 20+ делами.

Перед массовой публикацией словарь `data/reference/ddu_enum_dictionary.json` должен пройти проверку первой партией судебных актов.
