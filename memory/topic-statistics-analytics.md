# Статистика и аналитика по делам/выборкам

## Замысел
Сервисная статистика по загруженным судебным актам — отдельное продуктовое
направление для привлечения пользователей, в первую очередь профессиональной
аудитории (юристов). Связано с расширением концепции «широкого сервиса судебной
практики» (см. `memory/topic-positioning.md`, блок про профессиональную
аудиторию; раскрывает вопрос 6 из бэклога обсуждений там же).

## Что можно вытаскивать из текущего JSON v2
(поля уже есть в схеме, препон на уровне данных почти нет)

### Суммы (claims_and_result)
- По категориям: цена товара/услуги, штраф 50% по п.6 ст.13 ЗоЗПП, неустойка
  (ст.23/23.1/28), моральный вред (ст.15), убытки, судебные расходы.
- Метрики: диапазон min–max, медиана, среднее, квартили, распределение.
- Соотношение «заявлено / присуждено».

### Исходы дел
- Винрейт потребителя (% удовлетворения) — общий и в разрезах.
- Срез «тип требования × результат».

### Срезы / фильтры (группировки)
- По регионам, судам, инстанциям, судебным системам.
- По типу спора (dispute_type_code), типу требования (claim_type_codes), вертикали.
- По оппоненту (platform_or_company) — топ-ответчиков (Озон, ВБ, банки, застройщики).
- По периоду (год решения) — тренд во времени.

### Нормы и доводы (legal_analysis)
- Частота применяемых норм (какие статьи ЗоЗПП реально работают).
- Типичные отвергнутые доводы.

### Процесс (если есть даты в timeline)
- Длительность рассмотрения дела.

## Аудитории — что кому показывать
- Обычный пользователь: простые метрики на страницах-ситуациях («в N% случаев
  потребителю возвращают деньги», «типичный штраф — X ₽»).
- Юрист: granular-срезы, фильтры по судам/регионам/оппонентам, частоты норм,
  экспорт, отдельные аналитические страницы.

## Ключевые риски и ограничения
1. Минимальный размер выборки: при малых n региональные/судебные срезы вводят
   в заблуждение. Нужен порог (например, n ≥ 20–30 в срезе); ниже — не показывать
   или явно предупреждать.
2. E-E-A-T / YMYL: «статистика» при малых выборках — дезинформация. Обязательно
   указывать n, период, методологию, оговорку о нерепрезентативности.
3. SEO / индексация: массовая генерация срезов region × category × claim — риск
   «Scaled Content Abuse». Стратегия: единичные сильные статические лонгриды с
   уникальным текстом + интерактивные дашборды (JS, без отдельных индексируемых
   URL). Согласуется с правилом из technical-specification-mvp.md (служебные
   страницы, строки 88–94, 99–104).
4. Готовность данных: суммы часто в свободном тексте (нужна нормализация валюты);
   длительность рассмотрения требует дат подачи; «факторы успеха/провала»
   требуют качественной доп. разметки.
