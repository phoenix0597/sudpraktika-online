## Повторяемый контракт парсинга

- Контракт `data/review/_TASK_PARSE_NEXT_BATCH_AGY.md` переработан из задачи на конкретную партию в универсальный повторяемый регламент: агент сам находит незавершённый batch или создаёт новый `data/parse_batches/YYYY-MM-DD-zpp-next[-N]/`.
- Контракт теперь покрывает этапы, специфичные для парсинга: создание batch-папки, обновление реестра, поиск и дедупликацию кандидатов, скачивание raw-актов, постановку в очередь, структурирование, валидацию, отчёты и handoff через `batch_state.md`. Общий протокол восстановления контекста не дублируется.
- Стратегия парсинга универсализирована: агент перед каждой партией сам считает текущее покрытие по `taxonomy.dispute_type_code` и выбирает слабые/перспективные кластеры. Перечень кластеров признан открытым; новые повторяющиеся ЗоЗПП-ситуации фиксируются как кандидаты на новый кластер, но словарь не меняется без подтверждения пользователя.
- Пользовательская команда для любого агента должна быть короткой: «Восстанови контекст. Сделай парсинг очередной партии судебных актов». Дальнейшие инструкции агент берёт из проектной памяти и контракта.
