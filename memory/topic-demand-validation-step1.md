# Валидация спроса Шаг 1 — Wordstat по 15 кластерам

## Метод
Автоматизированный прогон 15 кластеров через Yandex Wordstat API (GA) — скрипт `scripts/wordstat_fetch.py`, методы GetTop (объём + родственные фразы) и GetRegionsDistribution (география). Сырой результат: `output/wordstat_step1.csv`. Лимит-менеджмент: 30 запросов из суточного бюджета 950.

## Главные выводы

### 1. Пилотный сегмент («технически сложные товары») — слабее ожидаемого
Конкретные товарные кластеры (телефон, ноутбук, авто, смартфон) дают **199/47/13/40 показов** — это очень узкий спрос. Даже якорная формулировка «возврат технически сложного товара ненадлежащего качества» = всего 361. Это **подтверждает риск**: прямой товарный интент узкий.

### 2. Спрос концентрируется на «процессуальных» темах, а не на товарах
Топ-4 кластера — **не конкретные товары, а инструменты защиты**: неустойка (5538), экспертиза (2078), интернет-магазин/маркетплейс (1547/1488). Это означает, что обыватель ищет не «как вернуть телефон», а **«как добиться возврата / посчитать неустойку / провести экспертизу»**. Структура страниц должна вести от процессуальной боли к делу, а не от товара к делу.

### 3. Маркетплейсы — подтверждённо приоритетная вертикаль (тренд)
«Возврат товара на маркетплейс(ах)» = 1488 + густая семантика с явными тренд-сигналами: «новые правила», «новый закон», «16 октября возврат товара на маркетплейсах». Это подтверждает решение из стратегии (Фаза 7) и, возможно, тянет на **более ранний приоритет**.

### 4. Неустойка — главный «инструментальный» спрос + готовый интерактив
Калькулятор неустойки (1206 показов отдельно) — это **прямой интерактивный блок** из ТЗ (страница-ситуация, блок 4). Высокий спрос + готовый функционал = сильная связка.

### 5. География: концентрация в кодах 225 / 3 / 1 / 40 / 213 / 59 / 17
Код 225 доминирует везде (Москва?), 3 — вторая, 1 — третья. Спрос концентрирован в нескольких регионах — важный сигнал для приоритизации контента/гео. Нужен маппинг кодов в названия (отдельная задача).

## Рекомендации для стратегии (кандидаты на закрепление)

- **Пересмотреть приоритизацию пилота:** товарный интент узок; сильнее спрос на процессуальные темы (неустойка, экспертиза) и маркетплейсы. Кандидат на обсуждение с пользователем: сместить акцент пилота с «конкретных товаров» на «процессуальные боли + маркетплейсы».
- **Неустойка + калькулятор** — готовый сильный интерактивный кластер (спрос 5538, калькулятор 1206).
- **Маркетплейсы** — подтвердить перенос из «Фаза 7» в более ранний приоритет.
- География требует маппинга кодов регионов в названия (через regions/tree).

## Открытые вопросы

- Подтвердить у пользователя пересмотр приоритизации пилота на основе этих данных.
- Донастроить dynamics (enum period + ISO-Timestamp) для оценки тренда — отдельно.
- Маппинг кодов регионов (225, 3, 1...) в названия.
