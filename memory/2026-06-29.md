## Стратегический разворот: сеть нишевых сайтов vs единый сервис

- Владелец меняет модель достижения трафик-цели 10–30K/сутки: вместо одного потенциально огромного сервиса — **сеть нишевых сайтов**, масштабируемая поэтапно (1 → 3–5 → 10+).
- Открыта новая дискуссия `memory/discussions/multi-site-network.md`.
- Зафиксированная рамка от владельца (через AskUserQuestion):
  1. Трафик-цель — **сумма по сети**, не на каждый сайт (каждый зрелый сайт ~1–3K/сутки).
  2. Степень изоляции сайтов — **открытый вопрос**, вынесен на обсуждение (реальная изоляция рисков vs footprint-риск).
  3. Выгоды, которые оптимизируем: изоляция рисков + уход от риска «крупного сайта» + тематическая специализация. Параллельные тесты — НЕ приоритет; стратегия последовательная.
  4. Деление — преимущественно **по правовым темам**, с региональными/судебными срезами внутри каждого сайта.
- Состав тот же: участники Zcode-GLM-5.2, Antigravity-Gemini-3.5-Flash, Zcode-Deepseek-v4-pro; судья Codex-5.5-xhigh.
- Предыдущая дискуссия (`content-scale-limits.md`) строилась на посылке «единый сервис» — теперь её рамка подлежит пересмотру по итогам этой.
- **Раунд 1 завершён:** даны ответы всех трёх участников. Zcode-Deepseek-v4-pro дал независимое мнение: рекомендация «сеть при жёстких условиях»; ключевой тезис — РСЯ-footprint неустраним, сеть выигрывает не столько изоляцией, сколько снятием с одного домена нереалистичной нагрузки и структурным соответствием праву. Главный непроверяемый риск — кросс-сайтовый penalty через общий РСЯ-аккаунт.
- Обсуждение принято владельцем и закрыто: thread переведён в `status: resolved`, substance промоутирован в `memory/topic-product-strategy.md` и `memory/topic-content-fill-plan.md`.
- Суть принятого решения: использовать модель сети нишевых сайтов при жёстких условиях; `sudpraktika-online.ru` — сайт №1 по ЗоПП, не головной портал сети; проверка условий РСЯ — блок-проверка №0; на старте допустим единый РСЯ только как осознанный компромисс с мониторингом и готовностью к разделению; новые сайты — последовательно, не чаще 1 сайта в 2–3 месяца, после гейтов 6–9 месяцев, индексации, трафика, отсутствия санкций и управляемой операционки.
