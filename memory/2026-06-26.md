## Дедупликация парсинга

- Принято MVP-решение: постоянный источник истины остаётся в raw/structured-файлах, быстрый индекс уже известных актов — `data/registry/case_registry.csv`, генератор — `scripts/build_case_registry.py`.
- `scripts/find_cases.py` по умолчанию исключает из новых поисковых CSV уже известные `docid`/URL; `--include-known` оставлен для аудитного режима. На старом `output/cases_ozon_regular.csv`: 87 уникальных URL, 50 известных отсекаются, 37 остаются новыми кандидатами.
- Масштабирование: CSV допустим для ближайшего этапа и десятков тысяч строк при загрузке в память; при настоящем массовом/параллельном парсинге постоянный слой — SQLite, Redis возможен только как runtime-очередь/локи, не как единственный источник истины. Подробности: `memory/topic-parsing-deduplication.md`.
