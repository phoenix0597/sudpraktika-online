# Доказательство ядра (Фаза 1.1) — LLM извлекает структуру акта

## Вердикт: ЯДРО РАБОТАЕТ

Главный риск проекта снят: LLM (DeepSeek v4-flash) качественно извлекает
структурированные данные из полного текста судебного акта по блокам из ТЗ.
Пайплайн «акт → структура» технически реализуем.

## Что протестировано

- **Акт:** дело № 2-80/2025, Усть-Коксинский районный суд (возврат видеокарты RTX 4090
  на Озон, спор о разнице цены). Тема точно из пилотной вертикали «Маркетплейсы».
- **LLM:** DeepSeek v4-flash, temperature=0, max_tokens=4000, json_object mode.
- **Промпт:** `scripts/prompt_extract.txt` — 8 обязательных блоков по ТЗ.
- **Скрипты:** `scripts/extract_act_text.py` (HTML→чистый текст),
  `scripts/extract_structure.py` (акт→JSON).

## A/B-тест DeepSeek vs эталон Gemini (Вопросы 1+2, 2026-06-24)

Пользователь предоставил эталонные ответы Gemini 3.5 flash на 2 вопроса по тому же акту
(пользовательская история + анализ судебной практики), полностью верифицированные как юристом.

**Результат:** DeepSeek v4-pro с правильными промптами выдал качество **на уровне или выше эталона**:
- Вопрос 1 (история): сравнимое качество, точнее по датам/суммам.
- Вопрос 2 (практика): глубже — найдены конкретные дела (Ап. опр. СПб горсуда № 2-664/2022),
  пункты Пленума ВС №17 (п.2, 12), солидарная ответственность по ст. 26.1+250-ФЗ (упущена Gemini).

**Вывод: менять модель на Gemini не нужно.** DeepSeek v4-pro с правильным промптом достаточен.
Ключ был в промпте (задача «расскажи/проанализируй» vs «извлеки поля»), не в модели.

Промпты-эталоны: `scripts/prompt_user_story.txt`, `scripts/prompt_practice_analysis.txt`.
Результаты: `output/test_user_story_ds.md`, `output/test_practice_ds.md`.

## Модель извлечения (закреплено 2026-06-24)
- **DeepSeek v4-pro** для качественного анализа (история, практика, holding).
- **DeepSeek v4-flash** для быстрого/дешёвого извлечения метаданных (номер дела, суммы, даты).
- Pro даёт глубину уровня верифицированного юристом-аналитика; flash — для масштаба.
